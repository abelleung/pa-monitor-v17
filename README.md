## 各文件功能详解

| 文件 | 功能概要 |
|------|----------|
| pa_monitor.py | **核心监控程序 v17.0**。中国平安(601318) T+0 四策略实盘监控：动量倒T / BOLL倒T / 冲高回落倒T + 正T。含信号触发、止盈止损、15:00收盘评估（最大价差）、心跳推送。从pytdx拉取1分钟K线，盘中持续运行。 |
| strategies.py | **策略判断函数 v17.0**。从 pa_monitor.py 解耦的四策略检测函数：check_momentum_sell_signal / check_boll_sell_signal / check_pullback_sell_signal / check_zhengT_buy_signal。 |
| indicators.py | **指标计算 + 策略参数 v17.0**。MACD/BOLL/涨跌/风险等技术指标计算，以及 STRATEGY_CONFIG 集中管理所有策略阈值。 |
| pa_notify.py | **推送通知模块**。支持 Bark(iOS)、ntfy(Android)、企业微信机器人三种渠道。强制不走代理，确保推送可靠。 |
| index.html | **前端页面**。暗色/亮色主题，展示实时K线、交易信号、统计数据，使用 JetBrains Mono + Noto Sans SC 字体。 |
| setup_monitor.sh | **管理脚本**。install/uninstall/start/stop/status/test/logs 子命令，配置 launchd + cron 自检。 |
| monitor_healthcheck.sh | **健康检查**。在关键时间点检查监控进程，心跳超时15分钟则重启。 |
| deploy.sh | **一键部署**。上传代码到云端服务器，更新服务版本号，重启服务。 |
| version_manager.sh | **版本管理**。backup/rollback/list/current 子命令，管理版本备份目录。 |
| test_v17_0.py | **回测脚本 v17.0**。遍历4种正T目标差价(0.20/0.30/0.40/0.50)和4种倒T目标差价(0.25/0.30/0.35/0.40)，用**触发后下一分钟~15:00**最大价差评估胜率。 |
| optimizer.py | **参数优化器**。分阶段穷举优化（Stage 1a-1d独立策略 + Stage 2联合 + Stage 3精细调优），输出最优参数组合。 |
| optimizer_multitarget.py | **多目标优化器**。针对不同目标差价（0.2/0.3/0.4/0.5元）分别优化，确保各差价下赢率>60%。 |

## 四种策略详解

### 策略总览

| 策略 | 方向 | 优先级 | 触发条件数 | 回测胜率(默认目标价) |
|------|------|--------|-----------|----------------------|
| 动量倒T | 卖出 | 第1优先 | 5个 | 80.0% @0.25元 |
| BOLL倒T | 卖出 | 第2优先 | 7个 | —（动量未触发时检查） |
| 冲高回落倒T | 卖出 | 第3优先 | 7个 | 71.7% @0.30元 |
| 正T买入 | 买入 | 独立 | 6个 + 2硬约束 | 75.0% @0.20元 |

> 倒T三策略互斥（同一时刻只触发一个），正T独立触发，倒T与正T互不阻塞。

---

### 1. 动量倒T（主力策略）

**方向**：卖出（做倒T，高位卖出低位买回）

**触发条件**（全部满足）：

| # | 条件 | 参数值 | 说明 |
|---|------|--------|------|
| 1 | 涨跌 > 阈值 | `涨跌 > 100` | 动量是核心，涨幅足够大 |
| 2 | 风险 > 阈值 | `风险 > 90` | v17.0优化，短期过热，回调概率高 |
| 3 | 成交额 ≥ | `≥ 3000万` | 有量才有价，防无量假突破 |
| 4 | 股价 > 日均价 + | `> 0.10元` | 价格在均线上方，确认强势 |
| 5 | \|MACD柱\| > | `> 0.06` | MACD动能确认 |

**限制条件**：
- 冷却期：触发后 **30根K线（30分钟）** 内不重复触发（`COOLDOWN_BARS=30`）
- 暴跌保护：日内跌幅 > **2.0%** 跳过倒T（`CRASH_DAY_DROP_MAX`）
- 时间窗口：**09:40 ~ 14:30**（交易时间内）

**止盈/止损**：
- 目标差价：正常日 **0.40元**，缩量日 **0.30元**，极端缩量 **0.15元**
- 止损差价：正常日 **0.30元**，缩量日 **0.20元**

**评估方式**（不影响状态）：
- 当天 **15:00收盘** 时，用触发后最大价差评估：`卖出价 - 触发后最低价（不含触发那根K线）`
- 评估窗口：触发后下一根K线 ~ 当天15:00
- 评估成功：`最大价差 ≥ 目标差价`
- 仅推送通知，不改变任何追踪状态

---

### 2. BOLL上轨倒T（补充策略）

**方向**：卖出（做倒T）

**触发条件**（全部满足，动量**未触发**时才检查）：

| # | 条件 | 参数值 | 说明 |
|---|------|--------|------|
| 1 | 最高价 ≥ BOLL上轨 × | `≥ 上轨 × 1.0` | v17.0优化，触碰BOLL上轨（容差0%） |
| 2 | 成交额 ≥ | `≥ 3000万` | 量能确认 |
| 3 | 收盘价 > 日均价 + | `> 0.10元` | 价格在均线上方 |
| 4 | 偏离日均价 > | `> 0.30元` | 深度超买 |
| 5 | MACD柱 > | `> 0.06` | MACD动能正向 |
| 6 | 当日涨幅 ≤ | `≤ 2.0%` | v17.0防追高 |
| 7 | 从当日最高回落 > | `> 0.10元` | 确认回落开始 |

**限制条件**：同动量倒T（30分钟冷却 + 暴跌保护 + 时间窗口）

**止盈/止损**：同动量倒T

**评估方式**：同动量倒T（15:00收盘最大价差评估）

---

### 3. 冲高回落倒T（低波动日备用）

**方向**：卖出（做倒T）

**触发条件**（全部满足，动量+BOLL**均未触发**且BOLL带宽**< 1.5元**时才检查）：

| # | 条件 | 参数值 | 说明 |
|---|------|--------|------|
| 1 | 盘中最高偏离均价 > | `> 0.20元` | 确认是冲高，非随机波动 |
| 2 | 从盘中最高回落 > | `> 0.15元` | v17.0收紧，确认回落 |
| 3 | 收盘价偏离均价 > | `> 0.20元` | 价格确实在均线上方 |
| 4 | 成交额 ≥ | `≥ 2000万` | v17.0优化（原4000万） |
| 5 | 收盘价 < BOLL上轨 | — | 已从上轨回落 |
| 6 | BOLL带宽 > | `> 1.5元` | 防窄幅震荡 |
| 7 | 时间在窗口内 | `09:40 ~ 14:00` | v17.0延长（原13:30） |

**限制条件**：同动量倒T（30分钟冷却 + 暴跌保护 + 时间窗口）

**止盈/止损**：同动量倒T

**评估方式**：同动量倒T（15:00收盘最大价差评估）

---

### 4. 正T买入（极低位高胜率）

**方向**：买入（做正T，低位买入高位卖出）

**触发条件**（全部满足）：

| # | 条件 | 参数值 | 说明 |
|---|------|--------|------|
| 1 | 均价 - 股价 > | `> 0.35元` | v17.0优化（原0.55元），深度超卖 |
| 2 | 风险 < | `< 12` | v17.0调整，超卖确认 |
| 3 | 成交额 ≥ | `≥ 5000万` | 强筛选器，有活跃资金 |
| 4 | 涨跌 ≤ | `≤ -1` | v17.0收紧，仍在下跌 |
| 5 | 日振幅 ≥ | `≥ 0.40元` | v17.0收紧，有反弹空间 |
| 6 | BOLL带宽 > | `> 0.1元` | 防极窄震荡 |

**硬约束**（不满足条件直接跳过）：
- 时间 < **14:00**（14点后胜率仅36%，时间不足）
- 连跌天数 ≤ **2天**（连跌>2天胜率骤降）

**限制条件**：
- 冷却期：触发后 **30根K线（30分钟）** 内不重复触发（`COOLDOWN_BARS=30`）
- 暴涨保护：日内涨幅 > **3.0%** 跳过正T（`BOOM_DAY_GAIN_MAX`）
- 时间窗口：**09:40 ~ 14:00**（14点后不触发）

**止盈/止损**：
- 目标差价：**0.25元**（固定，`ZHENGT_TARGET_DIFF`）
- 止损差价：**0.20元**（`ZHENGT_STOP_LOSS_DIFF`）

**评估方式**（不影响状态）：
- 当天 **15:00收盘** 时，用触发后最大价差评估：`触发后最高价（不含触发那根K线） - 买入价`
- 评估窗口：触发后下一根K线 ~ 当天15:00
- 评估成功：`最大价差 ≥ 目标差价`
- 仅推送通知，不改变任何追踪状态
- 正T买入后，**30分钟冷静期** 独立计算，不阻挡下次触发（无 `zhengt_buy_active` 阻塞）

## 技术栈

- **数据源**：pytdx（通达信行情）、腾讯分时接口
- **策略**：三策略倒T（动量/BOLL上轨/冲高回落）+ 正T极低位高胜率组合
- **后端**：Python 3.14 + pandas + matplotlib
- **推送**：Bark（iOS）、ntfy（Android）、企业微信Webhook
- **前端**：纯HTML/CSS/JS，无框架依赖
- **部署**：本地（macOS launchd）

## v17.0 回测结果（1-5月，94个交易日，触发后下一分钟~15:00评估）

### 评估逻辑说明
- 评估窗口：**触发后下一根K线 ~ 当天15:00**（不含触发那根K线本身）
- 正T最大价差 = 触发后窗口内最高价 - 买入价
- 倒T最大价差 = 卖出价 - 触发后窗口内最低价
- 与旧逻辑区别：旧逻辑用"当天全天"价格，会包含触发前的价格，虚高胜率

### 正T策略（32次触发）

| 目标差价 | 买入次数 | 成功次数 | 胜率 |
|---------|---------|---------|------|
| 0.20元 | 32 | 24 | **75.0%** ✅ |
| 0.30元 | 32 | 21 | **65.6%** ✅ |
| 0.40元 | 32 | 19 | **59.4%** ⚠️ |
| 0.50元 | 32 | 16 | **50.0%** ⚠️ 忽略 |

### 倒T策略（60次触发）

| 目标差价 | 卖出次数 | 成功次数 | 胜率 |
|---------|---------|---------|------|
| 0.25元 | 60 | 48 | **80.0%** ✅ |
| 0.30元 | 60 | 43 | **71.7%** ✅ |
| 0.35元 | 60 | 41 | **68.3%** ✅ |
| 0.40元 | 60 | 40 | **66.7%** ✅ |

### 关键发现

1. **正T 0.20元目标胜率75.0%**，优化后保持高胜率
2. **倒T 0.25元目标80.0%**，与历史回测（v10.2动量策略）一致
3. 正T 0.50元目标胜率降至50.0%，说明午后买入后涨超0.50元的场景不到六成
4. 倒T和正T独立运行，互不阻塞，30分钟冷却期自然过期
5. 参数优化后 Sharpe 比率达到 **6.02**（极优秀水平）
